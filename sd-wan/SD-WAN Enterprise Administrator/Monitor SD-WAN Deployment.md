### **Default Behavior**
- FortiManager shows device and interface statistics for the **past 10 minutes only**.
### **Enable Extended Monitoring**
- To display data over longer periods, enable history storage: 
```
config system admin setting 
	set sdwan-monitor-history enable 
end
```
- Once enabled, you can select time ranges (minutes, hours, days, or custom periods).
### **Data Retention**
- FortiManager stores SD-WAN device data for **up to 180 days** by default.
- To reduce retention and save disk space: 
```
  config system admin setting 
	  set rtm-max-monitor-by-days 
  end
```

### **Purpose**
- The **SD-WAN monitor template view** provides a **map-based visualization** of all devices linked to an SD-WAN overlay template.
### **Map Elements**
- **Hub devices**: Star icon
- **Spoke devices**: Device icon
- **Tunnels**: Lines connecting devices
### **Tunnel Status (Line Colors)**
- **Green**: Tunnel is up
- **Red**: Tunnel is down
### **Device Health (Icon Colors)**
- **Green**: All health checks pass
- **Orange**: Some health checks pass
- **Red**: All health checks fail
### **Device Position**
- Based on **longitude and latitude** entered during registration.
- If missing or device never connected: Position defaults to **0-0**.
### **Interactivity**
- **Hover**: Shows additional info
- **Click device/link**: Opens detailed pane with:
    - SD-WAN health check table
    - Underlay status table
    - Static & dynamic routing tables
    - IPsec VPN table

#### **Purpose of Analytics and Reporting**

- FortiAnalyzer relies on logs from devices for analytics and reporting.
- SD-WAN FortiView graphs use data from **health check SLA status logs** generated by FortiGate.
#### **Default Behavior**

- Health check SLA status logs are **not generated by default**.
- To enable full SD-WAN monitoring and reporting, these logs must be activated.
#### **How to Enable Logs**

- **On FortiGate:** Configure `sla-fail-log-period` and `sla-pass-log-period` in the **Performance SLA CLI**.
- **On FortiManager:** Configure these settings under **Advanced Options** in the **Performance SLA section**.
#### **Settings Explained**

- `sla-fail-log-period`: Interval (in seconds) for logs when a member **fails** SLA target.
- `sla-pass-log-period`: Interval (in seconds) for logs when a member **meets** SLA target.
- Default value: **0 seconds** (no logs generated).
#### **Example Configuration**

- Both settings set to **10 seconds**:
    - FortiAnalyzer receives SLA status logs every 10 seconds for each SD-WAN member with an SLA target.
    - Logs include detailed health and performance data.
